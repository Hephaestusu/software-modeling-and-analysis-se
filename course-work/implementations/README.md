Име на проекта: Discord – Online Community Platform

## Кратко описание

Проектът моделира опростена версия на платформа тип Discord за управление на сървъри, канали и потребители.  
Реализирани са:

- OLTP релационна база данни в Microsoft SQL Server (таблици User, Server, Channel, Role, Membership, Message, Emoji, Attachment, ChannelStats).
- Концептуален, логически, физически модел и модел на Data Warehouse (draw.io диаграми).
- Съхранена процедура за създаване на съобщение, потребителска функция за последна активност на потребител и тригер за поддръжка на статистика по канали.
- Набор от примерни SQL заявки за справки.
- Power BI отчет с визуализации върху данните (активност на потребители, сървъри, роли, канали и съобщения).

Структурата на проекта е в папка `Discord/` и е предназначена за изпълнение локално.

## Инсталация и стартиране

### 1. Необходими инструменти

- Microsoft SQL Server (Express или по-нова версия)
- SQL Server Management Studio (SSMS)
- Power BI Desktop
- Възможност за преглед на `.drawio` файлове (например чрез app.diagrams.net)

### 2. Създаване и попълване на базата данни

1. Клонирайте форка на репозиторието на курса и отворете папката  
   `course-work/implementations/Discord/SQL`.

2. В SSMS се свържете към вашия SQL Server и изпълнявайте скриптовете в следния ред:

   1. `1-Initialization.sql`  
      - създава база данни `Discord` и всички необходими таблици, ключове и връзки.

   2. `2-Functions.sql`  
      - създава потребителската функция `fn_UserLastActivity`.

   3. `3-Procedures.sql`  
      - създава съхранената процедура `CreateMessage`.

   4. `4-Triggers.sql`  
      - създава таблица `ChannelStats` (ако липсва) и тригера за актуализиране на статистика.

   5. `5-Insert_Data.sql`  
      - попълва базата с примерни данни (потребители, сървъри, канали, роли, съобщения и др.).

   6. (по желание) `6-DemoQueries.sql`  
      - съдържа примерни SELECT заявки за тестване на модела и данните.

### 3. Отваряне на Power BI отчета

1. Отворете папка `Discord/Power BI`.
2. Стартирайте файла `Discord-PBI.pbix` в Power BI Desktop.
3. Ако е необходимо, актуализирайте връзката към базата данни:
   - `Transform data` → `Data source settings` → променете сървъра/инстанцията към вашия локален SQL Server.
4. Натиснете `Refresh`, за да се заредят актуалните данни от базата `Discord`.

### 4. Диаграми и документация

- В папка `Discord/Discord-model-diagrams` се намират `.drawio` и `.png` файлове за:
  - Концептуален модел (Chen notation)
  - Логически модел (Crow’s Foot notation)
  - Физически модел (UML database notation)
  - Data Warehouse модел
- В папка `Discord/SQL` се намира файл `SQLdocumentation.docx` с подробно описание на модела, обектите в базата, функцията, процедурата, тригера и примерните заявки.
